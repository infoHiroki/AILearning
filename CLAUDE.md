## 📝 台本制作ルール

### 📝 基本的な制作順序
1. **元MD（教材ファイル）**: 基本となる学習コンテンツ
2. **スクリプト制作**: MD→`[ファイル名]_スクリプト.md`

### 📂 ファイル命名規則
```
元ファイル: プロンプト基礎.md
↓
スクリプト: プロンプト基礎_スクリプト.md
```

### 🎙️ 台本制作ガイドライン

#### 目的と用途
- **音声・動画制作用**: TTS変換・音声収録両方に対応
- **自然な話し言葉**: 聞きやすい口調に調整
- **分量管理**: 適切な文字数で読みやすい台本を作成

#### TTS台本作成ガイドライン

##### 1. 基本構造
```
みなさん、こんにちは。（導入挨拶）

今回は〇〇について学んでいきましょう。（テーマ提示）

それでは、まず最初に...（本編開始）

...

お疲れさまでした！（終了）
今日学んだことを、ぜひ実際に試してみてくださいね。（行動促進）
```

##### 2. 話し言葉調整ルール
- **自然な接続**: 「では」「それでは」「次に」等で滑らかに
- **聞き手を意識**: 「皆さん」「一緒に見ていきましょう」等
- **適切なポーズ**: 句読点で自然な間を作る
- **具体例重視**: 抽象的説明より実例を多用

##### 3. 自然な流れ構造
- **導入→展開→まとめ**: 基本的な話の流れに従う
- **段階的理解**: 基礎から応用へ順序立てて説明
- **効果**: 聞き手が迷わず内容を理解できる
- **実践**: 複雑な番号付けは避け、自然な接続詞で繋ぐ

##### 4. 避けるべき表現
- **専門用語連発**: 必ず分かりやすい言葉で補足
- **長すぎる文**: 1文は30秒以内で読める長さ
- **マークダウン記号**: `#` `**` `*` など技術記号は除去
- **URLや技術コード**: 音声では伝わらない情報
- **虚偽の企業事例**: ファクトチェックされていない架空の企業名・数値・成果は禁止
- **憶測の数値データ**: 根拠のない「○○%向上」「○日短縮」等の具体的数値は使用しない
- **不自然な驚き表現**: 「実は」「〜なんです」等の過度な驚き・発見演出は禁止

#### 🎯 台本準備の詳細ルール

##### 1. マークダウン記号の直接削除
- プログラムでクリーンアップせず、台本ファイルで修正
- 技術的負債を避ける（KISS原則）

##### 2. 見出し部分の句読点追加
- 元見出し（#, ##, ###）だった部分に句点「。」を追加
- 800ms程度の自然なポーズを作る
- 例：`はじめに` → `はじめに。`

##### 3. 列挙・項目説明での間改善
- 各項目説明後に読点「、」を追加
- 例：`1つ目は役割。` → `1つ目は役割、`
- 聞き手の理解を助ける適切なポーズを作成

##### 4. 例文・引用での間改善
- 導入部分に読点を追加
- 例：`良い例はこんな感じです。` → `良い例は、こんな感じです。`
- 引用文前後で自然な区切りを作る

##### 5. 接続詞での間改善
- 「そこで」「また」「最後に」等の後に読点追加
- 例：`そこで次は` → `そこで、次は`
- 話の流れをより自然にする

##### 6. XMLタグ・技術記号の音声変換
- XMLタグ: `<商品>` → `商品タグ開始` / `</商品>` → `商品タグ終了`
- 山かっこ: `<>` → 「山かっこ」「タグ」等の自然な表現
- コードブロック: 具体的な説明文に変換
- URL・記号: 音声で伝わる表現に変更

##### 7. 縦幅区切りの限界値設定
- 長すぎる段落は適切な位置で区切る
- 1段落最大10-15行を目安にする
- 話題転換や内容の切り替わりで自然に区切る
- 聞き手の集中力維持と理解促進を重視

#### 🔄 スクリプトの役割
- **スクリプト（_スクリプト.md）**: 動画制作用の自然な話し言葉台本
- **変換方向**: 元MD → スクリプト
- **最適化**: マークダウン記号除去、自然な話し言葉への調整

#### 📂 台本ファイル管理
- 元MDファイルと同階層に`[元ファイル名]_スクリプト.md`として作成
- 例：`プロンプト基礎.md` → `プロンプト基礎_スクリプト.md`

### 💡 二段階制作ワークフロー

#### 📝 第1段階：台本制作（このプロジェクト）
1. **元MDファイル選択**: 基本となる学習コンテンツを特定
2. **スクリプト作成**: 元MDから`[ファイル名]_スクリプト.md`を作成
3. **話し言葉調整**: マークダウン記号除去、句読点調整
4. **品質確認**: 音読テストで自然さを確認
5. **必要に応じて微調整**: より自然な表現に修正

#### 🎬 第2段階：動画制作（動画プロジェクト）
1. **スクリプトコピー**: `materials/[教材名]/[教材名]_スクリプト.md`に配置
2. **Googleスライド生成**: GAS + Clasp による自動スライド作成
3. **TTS音声生成**: Gemini TTSによる高品質日本語音声
4. **動画自動化**: Python + moviepy でスライド→動画変換
5. **最終出力**: Vimeo対応MP4として完成（約10分/15スライド）

### ⚠️ 重要な注意事項

- **台本に特化**: このプロジェクトではMarkdown台本の制作のみに集中
- **KISS原則**: シンプルで読みやすい台本を目指す
- **一貫性**: すべての台本で同じ品質基準を適用
- **外部連携**: 動画化は別プロジェクトで完全自動化実施
- **実績**: XML基礎で15スライド・10分動画の成功生成
- **効率化**: 25教材対応時間を12.5時間→50分に短縮

#### ⚡ アトミックコミットの原則

1. **単一責任**: 1コミット = 1つの論理的変更
   - ✅ 良い例: 台本制作ルールの追加のみ
   - ❌ 悪い例: 台本制作 + フォルダ整理 + ドキュメント更新

2. **独立性**: 各コミットが単独でテスト可能
   - 機能追加と設定変更は別コミット
   - ファイル移動と内容変更は別コミット

3. **完結性**: コミット時点でプロジェクトが動作する状態
   - 破綻した中間状態のコミットは避ける
   - 必要な依存関係も含めてコミット
