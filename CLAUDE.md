## 🎵 TTS（音声変換）制作ルール（2025年1月策定）

### 📝 制作フロー

#### 基本的な制作順序
1. **元MD（教材ファイル）**: 基本となる学習コンテンツ
2. **TTS台本制作**: MD→`[ファイル名]_TTS.md`

#### ファイル命名規則
```
元ファイル: プロンプト基礎.md
↓
TTS台本: プロンプト基礎_TTS.md
```

### 🎙️ TTS台本制作ルール

#### 目的と用途
- **Gemini TTS変換用**: TTS Converterでの音声生成用台本
- **自然な話し言葉**: 読み上げても聞きやすい口調に調整
- **時間管理**: 1本10-15分程度の音声コンテンツを想定

#### TTS台本作成ガイドライン

##### 1. 基本構造
```
みなさん、こんにちは。（導入挨拶）

今回は〇〇について学んでいきましょう。（テーマ提示）

それでは、まず最初に...（本編開始）

...

お疲れさまでした！（終了）
今日学んだことを、ぜひ実際に試してみてくださいね。（行動促進）
```

##### 2. 話し言葉調整ルール
- **自然な接続**: 「では」「それでは」「次に」等で滑らかに
- **聞き手を意識**: 「皆さん」「一緒に見ていきましょう」等
- **適切なポーズ**: 句読点で自然な間を作る
- **具体例重視**: 抽象的説明より実例を多用

##### 3. スリーポイント構造（3つのメッセージ法）
- **Rule of Three適用**: 人間が最も記憶しやすい情報単位が3つ
- **本質的メッセージの厳選**: 複雑な内容を3つの核心ポイントに絞る
- **構造例**: 
  ```
  第1のポイント：[核心メッセージ1]
  第2のポイント：[核心メッセージ2] 
  第3のポイント：[核心メッセージ3]
  ```
- **効果**: 聞き手の理解と記憶定着の向上
- **実践**: 元コンテンツから「これだけは伝えたい」3要素を抽出

##### 4. 避けるべき表現
- **専門用語連発**: 必ず分かりやすい言葉で補足
- **長すぎる文**: 1文は30秒以内で読める長さ
- **マークダウン記号**: `#` `**` `*` など技術記号は除去
- **URLや技術コード**: 音声では伝わらない情報
- **虚偽の企業事例**: ファクトチェックされていない架空の企業名・数値・成果は禁止
- **憶測の数値データ**: 根拠のない「○○%向上」「○日短縮」等の具体的数値は使用しない
- **不自然な驚き表現**: 「実は」「〜なんです」等の過度な驚き・発見演出は禁止

### 📝 サムネイル制作の基本原則

#### 🎯 技術仕様
- **サイズ**: 1280×720px（YouTube標準）
- **ファイル形式**: HTML + CSS（スクリーンショット用）
- **フォント**: Hiragino Kaku Gothic Pro, Noto Sans JP

    position: absolute;
    width: 100%;
    height: 100%;
    opacity: 0.3;
    z-index: 0;
}

    position: absolute;

- **文字間調整**: 個別設定で詰める

## 🎵 TTS（音声変換）ルール・ガイドライン

### 📝 TTS台本作成の基本原則

#### 🎯 台本準備ルール
1. **マークダウン記号は台本で直接削除**
   - プログラムでクリーンアップせず、台本ファイルで修正
   - 技術的負債を避ける（KISS原則）

2. **見出し部分の句読点追加**
   - 元見出し（#, ##, ###）だった部分に句点「。」を追加
   - 800ms程度の自然なポーズを作る
   - 例：`はじめに` → `はじめに。`

3. **列挙・項目説明での間改善**
   - 各項目説明後に読点「、」を追加
   - 例：`1つ目は役割。` → `1つ目は役割、`
   - 聞き手の理解を助ける適切なポーズを作成

4. **例文・引用での間改善**
   - 導入部分に読点を追加
   - 例：`良い例はこんな感じです。` → `良い例は、こんな感じです。`
   - 引用文前後で自然な区切りを作る

5. **接続詞での間改善**  
   - 「そこで」「また」「最後に」等の後に読点追加
   - 例：`そこで次は` → `そこで、次は`
   - 話の流れをより自然にする

6. **XMLタグ・技術記号の音声変換**
   - XMLタグ: `<商品>` → `商品タグ開始` / `</商品>` → `商品タグ終了`
   - 山かっこ: `<>` → 「山かっこ」「タグ」等の自然な表現
   - コードブロック: 具体的な説明文に変換
   - URL・記号: 音声で伝わる表現に変更

7. **縦幅区切りの限界値設定**
   - 長すぎる段落は適切な位置で区切る
   - 1段落最大10-15行を目安にする
   - 話題転換や内容の切り替わりで自然に区切る
   - 聞き手の集中力維持と理解促進を重視

#### 🔄 スクリプトとTTS台本の関係
- **スクリプト（_スクリプト.md）**: 動画・音声制作用の自然な話し言葉
- **TTS台本（_TTS.md）**: スクリプトをさらにTTS変換用に最適化
- **変換方向**: 元MD → スクリプト → TTS台本
- **TTS専用調整**: XMLタグ変換、記号除去、間調整の追加実施

##### ファイル管理ルール
1. **命名規則：タイムスタンプ（YYYYMMDD_HHMM）**
   - ❌ 悪い例：`プロンプト基礎_TTS_最終版.mp3`
   - ✅ 良い例：`プロンプト基礎_TTS_20250108_1430.mp3`
   - 「最終版」等の意味のない命名は禁止

2. **フォルダ構成**
   ```
   tts_converter/
   ├── audio_output/          # 生成音声ファイル
   │   ├── [元ファイル名]_YYYYMMDD_HHMM.mp3
   │   └── ...
   ├── tts_converter.py       # 変換プログラム
   └── main.py               # CLI interface
   ```

3. **台本ファイルの配置**
   - 元MDファイルと同階層に`[元ファイル名]_TTS.md`として作成
   - 例：`プロンプト基礎.md` → `プロンプト基礎_TTS.md`
